# AI Weaver — Landing Page

Static landing page deployed to GitHub Pages at **https://rrthurein.github.io/ai-weaver/**

## Files

| File | Purpose |
|------|---------|
| `final-real.html` | Working/source copy of the landing page — edit this one. |
| `index.html` | Deployed page. **Generated by `deploy.sh`** — do not edit directly. |
| `privacy.html` | Privacy policy page. |
| `screenshot/` | Production image assets. |
| `deploy.sh` | Build + deploy script (see below). |

## Deploying

Run:

```bash
./deploy.sh
```

The script performs these steps, in order:

1. **Rename** — `final-real.html` (or `final.html`) → `index.html`, overwriting the
   existing `index.html`.
2. **Fix links** — rewrites references so the deployed page uses clean paths:
   - `final-real.html#…` / `final.html#…` → `./#…`
   - `final-real.html` / `final.html` → `./`
   - `privacy-final.html` → `privacy.html`
3. **Fix image paths** — rewrites `assets/real-*.png` source paths to the matching
   files under `screenshot/`.
4. **Commit & push** — `git add -A`, commits with a timestamped message, and pushes
   to `origin main` (which triggers the GitHub Pages deploy).

### Notes / gotchas

- The script is **destructive and non-interactive**: it overwrites `index.html`, then
  commits and pushes immediately with no confirmation prompt.
- Because the source file is *renamed* (`mv`), `final-real.html` no longer exists after
  a deploy. Recover the source from `index.html` or git history if needed.
- Always edit `final-real.html`, never `index.html` directly — your changes to
  `index.html` will be overwritten on the next deploy.
